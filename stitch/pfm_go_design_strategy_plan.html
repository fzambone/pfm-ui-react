# PFM-Go Frontend Design Plan Based on the provided API documentation and the visual reference (UI.webp), I will create a high-fidelity, modern "glassmorphic" interface for the Personal Finance Manager. ## Visual Direction * **Aesthetic:** Glassmorphism (frosted glass, soft shadows, layering). * **Palette:** Dark mode base with vibrant accent gradients (orange, yellow, teal) as seen in the reference. * **Typography:** Clean, modern sans-serif (e.g., Inter or Roboto). * **Layout:** Sidebar navigation with a focused central workspace. ## Proposed Screens ### 1. Unified Dashboard (The "Home" View) * **Purpose:** At-a-glance household health. * **Key Elements:** * Household switcher (top bar). * Total Balance, Earnings, and Expenses summary cards. * Main spending/income chart (reimagining the wave chart from the reference). * Quick actions: "Post Transaction", "Transfer". * Recent activity list. ### 2. Accounts & Credit Cards * **Purpose:** Managing financial sources. * **Key Elements:** * Card-based account list showing balances (Checking, Savings, Investment). * Specialized Credit Card views showing limits, closing days, and due dates. * "Add Account" flow. * Balance reconciliation action. ### 3. Transaction Ledger * **Purpose:** Detailed history and data entry. * **Key Elements:** * Searchable/filterable list of all transactions. * Double-entry details (visualized simply as From/To). * Pagination controls. * New Transaction form (Modal or Sidebar). ### 4. Household & Profile Settings * **Purpose:** Admin and user management. * **Key Elements:** * Member management (Add/Remove members, role badges). * Household renaming. * User profile settings (Display name, password change). ### 5. Authentication (Login/Signup) * **Purpose:** Secure entry. * **Key Elements:** Sleek glassmorphic forms for email/password authentication. --- **Next Steps:** Once confirmed, I will create the Design System and begin generating these screens.
